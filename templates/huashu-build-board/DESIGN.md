# Huashu Build Board

Use for systems, benchmarks, methods, and post-training decks that need the
hard confidence of a build board: strong rules, large numbers, and no decorative
AI styling.

## Grammar

- Ivory/black base with one red signal color.
- Heavy sans headlines and oversized metric cells.
- Thick borders, ledger rows, and build-board rhythm replace cards.
- Grid and evidence scale do the design work.
- Avoid neon, purple gradients, glassmorphism, mock dashboards, and ornamental
  icons.

## Layout

- Cover: `cover-poster-grid`
- Evidence: `proof-ledger`
- Artifact content: `artifact-ledger`
- Matrix: `matrix-ledger`
- Metrics: `metrics-ledger`
- Labels: `content-field-manual`

## Image Rules

- Proof screenshots must be large enough to read in projection.
- Crop benchmark charts to the result region plus labels; do not show browser
  chrome unless it proves provenance.
- Metrics are allowed to be loud only when tied to source evidence.
- If the slide starts looking like a generic startup pitch, remove decoration
  and make the board denser.
