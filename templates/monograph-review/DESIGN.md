# Monograph Review

Use for public researcher profiles, paper talks, and project briefs that should
feel like an academic homepage plus printed review, not a slide generator.

## Sources

- PRISM: identity rail, selected publications, source-linked content, serif/sans
  pairing, configuration-driven academic structure.
- Information Architects: content-first hierarchy, no decoration, readable line
  length, link/source behavior as part of the design.
- Huashu critique rules: every visible box must have a job; remove any element
  that does not improve function.

## Visual Grammar

- Near-white academic paper with slate ink and muted gold only for metadata.
- Serif display headings, restrained sans body, mono kickers.
- Publication/project rows replace generic cards.
- Proof windows are flat, bordered, and captioned; no shadows unless needed for
  separation.
- Density target: useful 55-70% fill with strong source hierarchy.

## Slide Jobs

- Cover: source rail plus positioning statement and credibility anchors.
- Stack/content: claim and a readable source artifact, not a decorative thumbnail.
- Matrix: selected-work ledger with short failure -> artifact rows.
- Evidence: proof image dominates; side notes stay compact.
- Close: synthesis with source trail or links.

## Avoid

- Empty title pages, feature-card mosaics, purple gradients, glass, glow.
- Portraits as proof unless the slide is explicitly about identity.
- Summaries unsupported by homepage, paper, repo, benchmark, or project evidence.
