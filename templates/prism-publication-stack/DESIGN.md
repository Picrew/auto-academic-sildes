# Prism Publication Stack

Use for researcher, lab, or project profiles where selected work, papers,
benchmarks, and project artifacts need to read like a polished academic
homepage rather than a resume slide.

## Grammar

- Near-white page, slate ink, warm gold source cues.
- Left source/identity rail on covers; right side behaves like a publication
  index with concise claims.
- Publication/project rows are preferred over generic cards.
- Evidence slides use a dossier proof surface with readable caption and source.
- No glossy UI, no gradient hero, no fake dashboard decoration.

## Layout

- Cover: `cover-source-rail`
- Evidence: `proof-dossier`
- Artifact content: `artifact-dossier`
- Matrix: `matrix-map`
- Labels: `content-label-board`

## Image Rules

- Identity images should be large enough to anchor the person or project.
- Screenshots must be cropped to the inspected region and carry a caption/source
  line that names the visible artifact.
- If a source image only works as decoration, remove it or convert the slide to a
  text-led publication row.
