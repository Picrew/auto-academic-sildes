# Public Profile Deck Skill

Verify identity and work sources before writing. Use homepage/profile pages as identity anchors and project/paper/benchmark pages as proof surfaces.

Start from this spine only when it fits: cover, signature, contribution map, identity evidence, artifact evidence, synthesis.

Avoid overclaiming, decorative portraits, repeated template phrases, and internal critique language. Export through `uv run academic-deck html-pptx`.

After export, read `evidence-report.md` and check `Evidence Mix`. A public profile should pair identity evidence with at least one work artifact such as a project page, repo, paper/result figure, benchmark, or leaderboard.
